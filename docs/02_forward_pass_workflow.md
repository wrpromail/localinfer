# Qwen3.5-0.8B 整体计算与 Token 生成流程解析

正如您所察觉到的，单纯看权重的维度就像在看汽车零件的尺寸，如果不了解发动机的运转原理，就无法真正把车组装起来。
大语言模型的核心任务是：**“根据前面的所有词，预测下一个词的概率”**。

在自回归（Autoregressive）生成模式下，生成 10 个 Token 的完整生命周期可以分为两大部分：**Prefill（预填充阶段）** 和 **Decode（解码阶段）**。

---

## 核心流程：Token 是如何算出来的？

假设我们输入给模型的话是：“你好，”。

### 第一步：分词与词嵌入 (Tokenization & Embedding)
1. **分词 (Tokenization)**：通过 Tokenizer 将文本 “你好，” 变成一串数字 ID。假设变成了 `[108, 332, 15]`。
2. **词嵌入 (Embedding)**：利用前面在 safetensors 里存着的 Embedding 矩阵（形状为 `[248320, 1024]`），像查字典一样，把这 3 个数字 ID 转换成 3 个 1024 维的浮点数向量。
   - 此时我们的数据形状 (Shape) 是：`[Sequence_Length=3, Hidden_Size=1024]`。
   - 这就是我们模型的主干道（Residual Stream）起点。

### 第二步：24 层骨干网络的穿梭 (Transformer / Mamba Layers)
接下来，这 `[3, 1024]` 的矩阵会依次无缝穿过 24 个 Layer。
对于**每一层 (Layer N)**，数据都会经历以下处理：

#### 1. 前置归一化 (Pre-RMSNorm)
- 数据进入该层后，首先经过 RMSNorm（一种层归一化），把这 1024 个特征的数值尺度拉平，防止梯度爆炸。

#### 2. 注意力计算 (Attention) —— 获取上下文“知识”
这就是当前词去“看”前面所有词的过程。Qwen3.5 有两种层结构：
- **如果是全注意力层 (Full Attention, 传统 Transformer)**:
  - **Q, K, V 投影**：通过矩阵乘法，将 1024 维的数据投影出 Query (我要找什么)、Key (我有什么特征)、Value (我的实际内容)。
  - **RoPE (旋转位置编码)**：把 Token 的位置信息（我是第几个词）以旋转矩阵的方式注入到 Q 和 K 的前 25% 维度中。
  - **KV Cache 保存**：将算好的 K 和 V 保存到内存中备用。
  - **计算注意力分数**：`Score = Q × K^T`，然后过 Softmax。这一步决定了当前词需要分多少注意力给前面的词。
  - **加权求和**：`Output = Score × V`，得出整合了上下文信息后的新向量。
- **如果是线性注意力层 (Linear Attention / SSM)**:
  - 这里不用做暴力的 `Q×K` 矩阵乘法，而是通过 1D 卷积和隐藏状态机 (Hidden State) 更新，它的计算复杂度与文本长度是线性的。我们只需要更新状态机内存即可。

#### 3. 第一次残差相加 (Residual Add)
- `主干道数据 = 原本的主干道数据 + Attention 模块的输出`。

#### 4. 第二次归一化与 MLP —— 处理与记忆激活
- 数据再次经过 RMSNorm。
- 送入 **MLP (SwiGLU)** 结构：正如我们之前用代码抓取的，数据被 `up_proj` 和 `gate_proj` 放大到 3584 维，进行 SiLU 非线性激活和门控过滤，再由 `down_proj` 压缩回 1024 维。

#### 5. 第二次残差相加
- `主干道数据 = 主干道数据 + MLP 模块的输出`。

*-> 至此，数据顺利通过 1 层，紧接着用同样的方式穿过剩下的 23 层。*

### 第三步：最后的预测 (LM Head)
1. 从第 24 层出来的矩阵形状依然是 `[3, 1024]`。
2. 我们只需要取出**最后一个词**对应的向量（即第 3 个词的那个 1024 维向量）。
3. 经过最后一次 RMSNorm。
4. 将这个向量与词表权重矩阵（LM Head，由于 `tie_word_embeddings` 为 True，它其实就是第一步用过的 Embedding 矩阵）相乘。
   - `[1, 1024] × [1024, 248320] -> [1, 248320]`
5. 我们得到了一个长度为 248,320 的向量（Logits），代表词表中每个词是下一个词的“得分”。
6. 找出得分最高（Argmax）的那个词的 ID，假设是 889（代表“世”字）。

---

## 自回归循环 (Autoregressive Loop) 与 KV Cache

上面计算出“世”字的过程，叫做 **Prefill（预填充阶段）**。
接下来我们要连续生成 10 个 token，就进入了 **Decode（解码阶段）**。

**如果没有 KV Cache：**
我们要把“你好，世” 4 个词一起重新送入模型，从头开始算一遍 4 个词的 QKV，这会造成大量的重复计算。

**有了 KV Cache：**
1. 既然前面 3 个词的 K 和 V 我们已经在前面 24 层中存下来了，我们**现在只把新生成的第 4 个词（“世”）**送入模型！
2. 此时输入的形状极小：`[Sequence_Length=1, Hidden_Size=1024]`。
3. 当这 1 个词走到 Attention 层计算出它的 Q、K、V 时，我们将新的 K、V 追加到之前存好的 KV 缓存中。
4. 然后，用这 1 个词的 Q，去和缓存中**所有的** K 相乘（计算注意力分数），再提取所有的 V。
5. 一路走到最后，得到第 5 个词。
6. 重复这个过程 10 次，我们就高效地完成了 10 个 Token 的连续生成！

这就是整个大模型推理的完整生命周期。我们将用 Python 和 PyTorch 将上面的每一句话变成代码。
