# 线性注意力 (Gated Delta Network / Mamba 变体) 学习指南

在 Qwen3.5-0.8B 的架构中，75% 的层级采用了“非 Transformer”的架构。这种架构通常被称为 **线性注意力 (Linear Attention)**，而具体到这个模型，它使用了一种更为强大的变体：**Gated Delta Network (门控 Delta 网络)**，它与 Mamba 属于同一家族（State Space Models, SSM 状态空间模型）。

这份文档旨在梳理其核心运行机制，帮助您理解下一代大语言模型的物理引擎。

---

## 1. 核心困境：为什么我们需要它？

传统的 Transformer (Full Attention) 有一个致命弱点：**KV Cache 爆炸**。
当模型在处理第 N 个词时，它必须把前 N-1 个词的特征 (Key 和 Value) 全都拿出来，与当前的词 (Query) 做一次巨大的矩阵乘法。
- **复杂度**：计算量和内存占用随着文本长度呈平方级 $O(N^2)$ 爆炸。
- **痛点**：对于 256K 超长文本，KV Cache 会轻易撑爆甚至几张 A100 显卡的显存。

**线性注意力的破局点：恒定记忆矩阵**
线性注意力彻底抛弃了“保留所有历史特征”的做法。它引入了一个大小永远固定的**状态机/记忆矩阵 (RNN State)**。无论模型读了 10 个词还是 10 万个词，所有的历史知识都会被“压缩”进这个大小不变的盒子里。推理时的内存和计算量是绝对的 $O(1)$！

---

## 2. 核心组件解析

在我们的逆向代码中，Linear Attention 包含了以下几个核心特异性结构：

### 2.1 一维深度卷积 (`conv1d`)
*   **物理意义**：在状态机进行全局知识融合前，给当前特征提供一个“短时局部记忆”。
*   **机制**：`kernel_size=4`，意味着当前输入的词，会先跟前面 3 个词进行一次滑动加权。这弥补了纯 RNN 结构在处理“连词成短语”时的局部特征提取弱势。

### 2.2 数据驱动的门控机制 (Data-dependent Gating)
传统的 RNN (如 LSTM) 也有遗忘门，但在大语言模型中，这种遗忘机制被推向了极致：
*   模型拥有 `in_proj_a` 和 `in_proj_b` 两个投影层，专门用于计算**当前这个词**的门控系数。
*   **$g_t$ (衰减门)**：由 `A_log` 参数、偏置 `dt_bias` 和输入特征共同计算得出。它决定了记忆矩阵中的旧知识应该被衰减掉多少。
*   **$\beta_t$ (步长/缩放门)**：决定了当前词带来的新知识应该以多大的比例融入记忆矩阵。

### 2.3 Delta 更新法则 (核心数学引擎)
这是整个架构皇冠上的明珠。有别于传统注意力 $Softmax(Q \cdot K^T) \cdot V$，它的更新法则更像是物理学中的动量更新：

1.  **老记忆衰减**：`rnn_state = rnn_state * exp(g_t)`
2.  **新知识融合**：计算当前词的新知识，并用 $\beta_t$ 缩放。这里的本质是利用 Delta Rule 进行修正（用当前值减去模型原本已有的预期值，再加入状态）。
3.  **最终提取**：当前的查询 $Q_t$ 去乘以更新后的 `rnn_state`，提取出有用的信息作为最终输出。

---

## 3. 并行与串行的双重人格 (Duality)

您可能会问，既然它是像 RNN 一样一个词一个词更新记忆矩阵，那训练大模型的时候岂不是慢如蜗牛？
这就是它最神奇的地方——**双重人格**：

*   **RNN 模式（生成阶段）**：即我们 `08_generate.py` 中使用的模式。每吞吐一个词，就做一次 $O(1)$ 的状态更新。显存小，速度极快。
*   **并行扫描模式（训练 / Prefill 阶段）**：数学家们发现，上面复杂的衰减更新公式，在数学上严格满足**结合律**。这意味着，利用 GPU 多线程，可以设计出类似“并行前缀和 (Parallel Associative Scan)”的高级算法，一口气把成千上万个词的状态机更新全部并行算完，速度不亚于传统 Transformer！

---

## 4. 工程与硬件视角

在纯 Python/PyTorch 代码中实现它时，会遇到极大的挑战：
1.  **数值极度敏感**：衰减门包含指数运算 `exp(g_t)`，在数百个词的连乘更新下，极微小的舍入误差会导致数值爆炸 (NaN) 或坍缩，直接使得模型输出乱码（比如重复单个无意义符号）。
2.  **CUDA/Triton 底层依赖**：为了解决上述问题，并且发挥出并行扫描的威力，业界（如 Hugging Face, FlashAttention, vLLM 等）都会使用极致优化的底层 C++/CUDA 内核（如 `causal_conv1d` 和 `flash_linear_attention`），将所有的运算融合在 GPU 的 SRAM (高速缓存) 中，确保精度不出错且跑满带宽。

**总结：** 掌握了这种架构，您就把握住了大模型从 Transformer 向更高效率进化的未来命脉。它用精妙的控制论（门控与状态更新）换取了对计算资源的极大解放。
